# AGENTS.md

## Fluxo de entrega

- Após cada alteração concluída e validada, faça `git add`, `git commit` e `git push`.
- Não deixe mudanças prontas apenas no working tree, a menos que o usuário peça explicitamente para não publicar.
- Antes de fazer push, sempre validar com `npm run build`.
- Se houver mudanças não relacionadas já pendentes, não misture sem confirmar antes.

## Escopo

- Estas regras valem para o repositório `study_app` neste diretório.
